COURSE MAP

本节只讲三种真正常用的位置

场景人在哪里介入适合什么时候用
Node 级 HITL自定义 StateGraph 的某个节点里 interrupt()你需要完整控制 state、router、审计日志
Agent 级 HITLAgent 准备调用 tools 前 interrupt_before=["tools"]你用预构建 Agent，但要拦住危险工具调用
Wrapper 级 HITL把危险 tool 包一层，在 wrapper 里 interrupt()多个 Agent / Graph 都会调用同一个危险能力
核心判断：业务流程要精细控制，用 Node；工具调用要统一拦截，用 Agent；危险能力要全局安全，用 Wrapper。

NODE LEVEL HITL

Node 级：LLM 生成内容，节点里决定要不要等人

为什么先讲它

这是最白盒的做法。你自己写 StateGraph，所以 state、节点、router 和审计日志都能被学生逐行看见。

这里要讲清楚：风险识别不等于风险拦截。

Notebook 主线

输入发布请求
→ LLM 生成文案
→ LLM / 规则做风险审查
→ 高风险暂停给人
→ 人回复
→ 继续发布或拒绝

STATE & PROMPTS

教学重点：让状态和 prompt 都被学生看见

State 保存业务事实

  • generated_draft：LLM 生成的文案
  • risk_level / risk_score：审查结果
  • review_decision：人工回复
  • final_content / publish_status：最终结果
  • audit_log：节点动作链路

Prompt 是课程材料

文案生成提示词
→ 只负责生成 draft

风险审查提示词
→ 只负责返回 risk_level、risk_score、risk_reasons、needs_human_review

AGENT LEVEL HITL

Agent 级：Agent 想调用工具前，先停下来给人看

人类场景

用户说：帮我发布这条内容
Agent 思考：我需要调用 publish_to_channel
系统暂停：这个工具会真的发布，先给人确认
人确认后：工具才执行

落地方式

create_agent(
  model=agent_model,
  tools=[publish_to_channel],
  checkpointer=MemorySaver(),
  interrupt_before=["tools"],
)

这种方式拦的是 Agent 即将调用工具这个动作。

Agent 级 HITL8 / 15
AGENT EVIDENCE

Agent 停住时，人看到的是 tool name 和 tool args

Agent 想做什么

{
  "name": "publish_to_channel",
  "args": {
    "content": "...",
    "channel": "公众号"
  }
}

为什么停住

  • tools 节点会执行真实发布动作。
  • interrupt_before 让工具调用前先暂停。
  • 人要判断是否允许这个 tool_call 执行。
适合预构建 Agent：你不想重写整张图，只想统一拦住工具调用。
运行证据9 / 15
WRAPPER LEVEL HITL

Wrapper 级：危险能力自己带审批

为什么需要 wrapper

Agent 级拦截依赖调用方真的进入 tools 节点。生产里更稳的做法是：危险能力自己先请求人工确认。

同一个发布能力被多个 Agent / Graph 复用时，审批边界应该放在能力入口。

执行模型

调用方准备发布参数
→ wrapper 先 interrupt
→ 人看 tool args
→ approve 后才调用真实 publish_service

运行证据11 / 15
CHECKPOINT

为什么 checkpointer 还是必须存在

真实审批不会一直挂着调用栈

审批可能几分钟后回来，也可能第二天才回来。系统不能让请求、线程、连接和进程一直占着资源等人。

更合理的机器动作

运行到风险点
→ 生成待办
→ checkpoint 保存 thread、暂停点、state/messages
→ 当前请求结束，释放资源
→ 人提交决定
→ Command(resume=...) 恢复
checkpoint 不是个性化记忆，而是保存可恢复现场，让流程可以断开后再继续。
恢复基础设施12 / 15
PITFALL

不要把 interrupt() 塞进一个大节点里

危险写法

生成文案
→ 调外部 API
→ 写数据库
→ interrupt 等人
→ 发布

恢复时，LangGraph 会从当前节点开头重新执行。

课程推荐拆法

生成文案节点
→ 风险审查节点
→ 人工审批节点
→ 发布节点

人工审批节点只准备 payload、接收 resume、写回 state。

SUMMARY

这节课真正要记住什么

Human-in-the-loop 是给 Agent 系统加一条安全边界：模型可以生成、判断、计划、准备 tool_call，但高风险动作真正发生前，必须有人确认。

你需要选择
完整业务状态和路由Node 级 HITL
拦住预构建 Agent 的工具调用Agent 级 HITL
让危险工具全局不漏审Wrapper 级 HITL
