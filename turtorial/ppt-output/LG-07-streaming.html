LangGraph 渐进式教程3 / 10
核心内容
values 看全局，updates 看变化

stream_mode="values" — 完整状态快照

{
  "type": "values",
  "ns": (),           # 命名空间
  "data": {         # 完整的 State
    "messages": [...],
    "outline": ["引言", "背景"],
    "current_node": "outline_planner"
  }
}

每次推送都是完整 State，直接替换即可。State 大时带宽消耗高。

stream_mode="updates" — 增量更新

{
  "type": "updates",
  "ns": (),
  "data": {
    "outline_planner": {
      "outline": ["引言", "背景"]
    }
  }
}

只推送节点返回值，数据量小。前端需要 merge 到现有 State。

vs 对比

values 适合 State 小、需要完整视图的场景；updates 适合 State 大、带宽敏感的场景（如移动端）。并行节点执行时，updates 会分开发送每个节点的结果。

核心内容
custom 看进度，tasks 看生命周期

stream_mode="custom" — 自定义事件

async def research_node(state, writer):
    sources = ["财经数据", "新闻", "竞品"]
    for i, source in enumerate(sources):
        writer({
            "type": "progress",
            "source": source,
            "percent": (i + 1) / len(sources) * 100,
            "status": "searching"
        })
        await search(source)

writer 是自动注入的——只要在 Node 函数签名中声明 writer 参数，LangGraph 就会自动传入。

stream_mode="tasks" — 任务生命周期

# Task Start
{"type": "tasks", "data": {
    "id": "outline_planner-abc",
    "name": "outline_planner",
    "input": {...},
    "triggers": ["__start__"]
}}

# Task Finish
{"type": "tasks", "data": {
    "id": "outline_planner-abc",
    "name": "outline_planner",
    "error": None,
    "result": {"outline": [...]}
}}

自动追踪 start/finish，无需在 Node 内手动 yield。失败时 error 字段包含完整错误信息。

LangGraph 渐进式教程6 / 10
核心内容
组合多种 mode，同时实现打字机 + 进度条 + 监控
async for mode, chunk in graph.astream(
    input,
    stream_mode=["messages", "custom", "tasks"],
    subgraphs=True    # ← 多 Agent 嵌套时必须开启
):
    if mode == "messages":
        msg, metadata = chunk["data"]
        append_token(msg.content)      # 打字机效果
    elif mode == "custom":
        update_progress(chunk["data"]["percent"])  # 进度条
    elif mode == "tasks":
        highlight_node(chunk["data"]["name"])   # 节点高亮

ns 命名空间含义

  • () = 顶层图
  • ("subgraph",) = 名为 subgraph 的子图
  • ("supervisor", "weather_expert") = supervisor 图中的 weather_expert 子图

前端处理策略

  • 顶层图事件 → 主面板显示
  • 子图事件 → 可折叠详情面板
  • 多层嵌套 → 按层级分组展示
LangGraph 渐进式教程7 / 10
核心内容
三段式流式协议：Start → Processing → End

start

占位，创建消息容器，插入进度条

processing

内容增量，追加到容器，更新打字机

end

收尾，移除光标，启用复制，触发后续

场景是否流式理由
LLM 生成自然语言必须流式消除等待焦虑、提升 perceived speed
思考过程/推理链建议流式让用户看到"AI 在思考"，建立信任
工具调用参数非流式结构化数据，一次性展示更高效
工具执行结果非流式结果完整性要求高，片段无意义
错误信息非流式错误需要立即、完整呈现

    const msg = JSON.parse(event.data);
    switch (msg.type) {
        case "token":      document.getElementById("article").textContent += msg.content; break;

阶段小结
核心要点回顾

5 种核心 Stream Mode

  • values = 完整状态快照——适合状态面板
  • updates = 增量更新——适合带宽敏感场景
  • messages = token 级流式——适合打字机效果
  • custom = 自定义事件——适合进度条
  • tasks = 任务生命周期——适合监控

关键技巧与注意事项

  • 组合 mode 时返回 (mode, chunk) 元组，需要解包
  • 多 Agent 嵌套必须设置 subgraphs=True
  • 通过 ns 命名空间区分事件来源
  • messages 的 token 用 += 追加，不是替换
  • 三段式协议：start → processing → end
"values 看全局，updates 看变化，messages 看逐字，custom 看进度，tasks 看生命周期，subgraphs=True 看嵌套。"
