## 二、统一视觉风格规范

所有图表遵循 **Tokyo Night** 主题风格，确保与 PPT 浑然一体：

### 色彩系统
```
背景色：     #0f172a  （深蓝黑，与幻灯片底色一致）
主文字：     #e2e8f0  （浅灰白）
次要文字：   #94a3b8  （灰蓝）
强调色：     #38bdf8  （亮蓝，用于核心节点/高亮）
成功/正面：  #9ece6a  （绿色）
警告/人工：  #e0a068  （橙黄）
错误/风险：  #f7768e  （粉红）
卡片背景：   #1e293b  （暗蓝灰，带 80% 不透明度）
边框线：     rgba(148,163,184,0.2)  （细灰蓝线）
```

### 设计原则
1. **深色底**：所有图表使用 #0f172a 作为背景，无缝融入幻灯片
2. **扁平化**：不使用渐变阴影，保持现代简洁
3. **圆角卡片**：节点使用 8px 圆角矩形，与 PPT 卡片风格一致
4. **字体**：中文用 Noto Sans SC / 英文用 Inter 或 JetBrains Mono
5. **连线**：使用 2px 实线箭头，颜色 #64748b，转折处圆润
6. **留白**：图表四周留出 40px 安全边距，避免贴边
7. **分辨率**：推荐 1920×1080 或 1600×900（16:9），导出 PNG 透明背景或深色底

### 图表类型与用途
| 类型 | 用途 | 风格 |
|------|------|------|
| 架构图 | 展示系统分层、模块关系 | 水平/垂直分层，卡片式节点 |
| 流程图 | 展示执行顺序、分支逻辑 | 箭头连接，条件分支用菱形 |
| 对比图 | 展示 A vs B 的差异 | 左右分栏，颜色区分 |
| 时序图 | 展示时间轴上的交互 | 水平时间线，阶段标注 |
| 树形图 | 展示层级结构 | 根节点在上，分层展开 |

## 三、逐章图表需求与提示词

> **提示词使用原则**：下面每段提示词可以直接复制到图片生成工具。提示词本身允许使用英文描述构图，因为多数模型对英文构图指令更稳定；但**画面中实际出现的文字必须以中文为主**。除 LangGraph、State、Node、Reducer、ToolNode、Checkpoint、Store、Cache、MCP、ReAct、Trace、Span、Event、Redis、PostgreSQL、Langfuse、Studio、DeepAgents、API、CLI、JSON、YAML、函数名、参数名等专有名词或代码标识符外，不要在图里出现英文说明。
>
> **出图硬约束**：不要把 `Background`、`Node style`、`Label`、`Color`、`Resolution` 等提示词说明画进图片；它们只是生成指令。所有标题、图例、注释、优缺点、场景说明都用中文。

## 四、批量生成建议

### 工具推荐
| 工具 | 适合场景 | 优势 |
|------|----------|------|
| **Mermaid.js + mermaid-cli** | 流程图、时序图、状态图 | 代码驱动，可版本控制 |
| **Excalidraw** | 架构图、手绘风格 | 自由绘制，导出 PNG |
| **PlantUML** | 复杂时序图、类图 | 文本驱动，专业规范 |
| **Draw.io (diagrams.net)** | 通用图表 | 功能全面，免费 |
| **GPT-4o / Claude** | 概念验证图 | 快速生成草稿 |

### 批量生成策略
1. **统一模板**：先用任意工具制作 1 张参考图，确定颜色、字体、间距
2. **分类生成**：
   - 流程图类（1-1, 2-1, 3-2, 5-1, 5-2, 6-1, 8-2）→ 用 Mermaid
   - 架构图类（1-2, 2-2, 4-1, 4-2, 8-1, 8-4, 9-3）→ 用 Excalidraw/Draw.io
   - 对比图类（2-3, 3-1, 4-3, 5-3, 6-2, 7-1）→ 用 Figma/Draw.io
   - 时序图类（3-2, 7-2）→ 用 Mermaid
   - 树形图类（9-1, 10-2）→ 用 Mermaid 或手动绘制
3. **命名规范**：`{章节}-{序号}-{描述}.png`，如 `LG-01-01-weatherbot-flow.png`
