# html-ppt-skill 功能全景分析

> 定位：HTML PPT Studio + 数据分析与办公提效全能助手
> 覆盖：数据处理 → 分析洞察 → 报告撰写 → 数据可视化 → PPT 制作的端到端工作流

---

## 一、核心能力矩阵（5大模块）

| 模块 | 能力 | 输出形态 | 典型场景 |
|------|------|---------|---------|
| **静态 HTML PPT** | 演讲/技术分享/逐字稿演示 | `.html` 文件，浏览器打开 | 技术分享、演讲、培训 |
| **HTML → PPTX** | 传统 PowerPoint 文件生成 | `.pptx` 文件 | 客户提案、需 Office 编辑的场合 |
| **交互式 HTML 报告** | 数据分析 + ECharts 图表 | `.html` 报告，可导出 PDF | 数据复盘、周报月报、投放分析 |
| **Excel 数据处理** | 读取/分析/转换 Excel | Markdown / CSV / JSON | 数据清洗、快速洞察 |
| **公式与测算** | ROI 计算、投放公式生成 | 公式文本 + 计算逻辑 | 投放优化、财务测算 |

## 二、静态 HTML PPT Studio（核心资产）

### 2.1 主题系统（36 套）
- **Developer 系列**：tokyo-night, dracula, catppuccin-mocha, terminal-green, nord
- **Professional 系列**：minimal-white, corporate-clean, academic-paper, swiss-grid
- **Creative 系列**：soft-pastel, memphis-pop, rainbow-gradient, neo-brutalism
- **Atmospheric 系列**：sunset-warm, aurora, vaporwave, cyberpunk-neon
- **Social/Mobile 系列**：xiaohongshu-white, japanese-minimal
- **特殊用途**：blueprint（架构图）、pitch-deck-vc（融资路演）、news-broadcast（新闻）

**主题选择机制**：3 个问题 → 2-3 个推荐
1. 受众是谁？（工程师 / 高管 / 客户 / 学生 / 投资人）
2. 调性是什么？（严肃 / 活泼 / 技术 / 前卫 / 温暖 / 极简）
3. 交付方式？（投影仪 / 屏幕共享 / 自读 / 手机 / 打印）

### 2.2 布局系统（31 种）
按叙事角色分类：

| 叙事角色 | 布局文件 | 用途 |
|---------|---------|------|
| **Hook** | cover.html, image-hero.html, big-quote.html | 封面、震撼开场 |
| **Context** | toc.html, mindmap.html | 目录、背景铺垫 |
| **Build** | bullets.html, two-column.html, three-column.html, flow-diagram.html, arch-diagram.html, architecture-focus.html, pipeline-focus.html, layer-stack-focus.html, sequence-flow.html, code.html, terminal.html | 论证、解释、架构讲解 |
| **Proof** | stat-highlight.html, kpi-grid.html, table.html, chart-bar.html, chart-line.html, chart-pie.html, chart-radar.html, image-grid.html | 数据证明、KPI 展示 |
| **Plan** | timeline.html, roadmap.html, gantt.html, process-steps.html, todo-checklist.html | 路线图、时间线、计划 |
| **Contrast** | comparison.html, diff.html, pros-cons.html, chart-radar.html | 前后对比、优劣分析 |
| **Close** | cta.html, thanks.html | 行动号召、致谢 |
| **Transition** | section-divider.html | 章节过渡 |

**认知负荷规则（5/5/5）**：
- 每行标题 ≤ 5 个词
- 每页 ≤ 5 个 bullet
- 连续文字页 ≤ 5 页，必须穿插视觉页

### 2.3 动画系统（47 种）
- **27 CSS 动画**：fade-up, fade-left, fade-right, rise-in, blur-in, zoom-pop, counter-up, stagger-list 等
- **20 Canvas FX**：particle-burst, confetti-cannon, firework, starfield, matrix-rain, knowledge-graph, neural-net, constellation, orbit-ring, galaxy-swirl 等

**动画叙事规则**：
- Hook → rise-in / blur-in + 最多 1 个 ambient FX
- Build → fade-up / fade-left / fade-right / stagger-list
- Proof → counter-up / zoom-pop
- Compare → 从相反方向进入
- Close → 更平静、更简单的动画

### 2.4 演讲者模式（Presenter Mode）
- **触发**：S 键打开独立弹出窗口
- **4 张磁吸卡片**：CURRENT（当前幻灯片）、NEXT（下一张）、SPEAKER SCRIPT（逐字稿）、TIMER（计时器）
- **特性**：iframe 像素级预览、拖拽调整、postMessage 同步、BroadcastChannel 双窗同步
- **逐字稿规则**：每页 150–300 字、加粗核心词、口语化表达

## 三、数据分析和报告能力

### 3.1 输出格式决策树
```
用户意图分析
├── 分析/报告/可视化 → 交互式 HTML 报告（默认）
├── 演讲/技术分享/逐字稿 → 静态 HTML PPT
├── 需要 .pptx 文件 → HTML → PPTX
└── 快速看数字 → Terminal + Markdown
```

### 3.2 数据报告风格库（5 种）
| 风格 | 标志元素 | 适用场景 |
|------|---------|---------|
| Financial Times | 三文鱼粉底 + 蓝色顶线 + 衬线标题 | 金融分析 |
| McKinsey Consulting | 深蓝 Header + Exhibit 编号 + 结论式标题 | 战略分析 |
| The Economist | 红色细条 + editorial 标题 + 杂志密度 | 行业洞察 |
| Goldman Sachs | Rating 徽章 + 金色强调 + 密集表格 | 财务建模 |
| Swiss / NZZ | 黑白灰红 + 72px 大字 + 极端字号对比 | 数据展示 |

### 3.3 分析哲学
- **结论先行** — 先说好坏，再说原因
- **数据说话** — 每个观点有数据支撑
- **具体可执行** — 建议能直接落地
- **不说废话** — 删掉「总而言之」等虚词

### 3.4 分析输出结构
```
核心结论（1-3 句，管理层只看这段）
→ 数据支撑（具体数字、对比、趋势）
→ 异常/风险
→ 可执行建议（3-5 条，按优先级）
→ 下一步（还能深挖什么）
```

### 3.5 投放/广告分析领域知识
- ROI 公式体系
- 投放维度拆解
- 行业法则和启发式判断

## 五、设计系统 DNA

### 5.1 核心追求
- **温暖专业感**：暖色调（奶油、珊瑚、暗金），像设计精良的杂志
- **信息优先**：设计服务于数据，每个元素帮助理解
- **10 米可读**：标题占 15-30%，辅助文字 ≥ 10pt，表格斑马纹
- **数据诚实**：Y 轴从 0 开始，绝对比例，极小值保护

### 5.2 明确避免
- 赛博霓虹 / 深蓝底 / 紫色底 / 纯黑纯白
- CDN 依赖（Playwright 离线截图白屏）
- CSS absolute 定位数据点（精度丢失）
- 同系列报告视觉不一致
- flex:1 大面积空白
- 金色文字在白底（对比度不足）

## 六、叙事和创作规则

### 6.1 断言标题规则
- **好**："缓存将延迟降低了 40%。"
- **坏**："性能"

### 6.2 轻量检查点
1. **大纲确认**：输出表格（# / Slide title / Narrative role / Layout / Notes），用户确认后再写 HTML
2. **样张确认**：生成 1-2 页样张，确认风格后再生成完整 deck

### 6.3 创作铁律
- Always start from a template（不从零写 slide）
- Plan reveal beats before you animate（先定论点，再加动画）
- Use tokens, not literal colors（用 CSS 变量，不用硬编码色值）
- Don't invent new layout files（优先组合现有 31 种布局）
- NEVER put presenter-only text on the slide itself（演讲者备注只能放 `.notes`）

## 九、一句话总结

**html-ppt-skill = 专业级静态 HTML 演示引擎 + 数据分析报告工厂 + PPTX 转换器**

覆盖从「Excel 原始数据」到「演讲级 HTML PPT」或「可编辑 PPTX」的完整链路，强调：
1. **叙事驱动**（每页一个断言，故事弧线清晰）
2. **认知负荷管理**（5/5/5 规则，图文交替）
3. **温暖专业感**（暖色调、信息优先、数据诚实）
4. **演讲友好**（演讲者模式、逐字稿、键盘导航）
