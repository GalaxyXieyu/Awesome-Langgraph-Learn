Layers · 分层聚焦

把系统看成一层一层搭起来，而不是一张平面图

L1

Experience Layer

用户界面、交互工作台、展示和输入行为。

L2

Control Layer

API、任务调度、权限、状态流转和执行控制面。

L3

Runtime Layer

图编排、工具、Writer、Prompt 和上下文运行时。

L4

State Layer

持久化、缓存、向量索引、观测和记忆语义。

Focused explanation

为什么 Control Layer 通常是平台系统最容易被低估的一层？

因为很多项目只强调“前端”和“Agent”，但真正决定系统可运营性的，往往是控制层：它负责请求承接、任务生命周期、权限约束、状态恢复和对前端的反馈协议。

Task lifecycle创建、排队、执行、失败、恢复、完成。
Policy gate权限、确认点、执行边界和风险控制。
Streaming bridge把中间步骤和最终结果持续送回 UI。
Operational clarity给日志、状态、观测和调试提供锚点。
