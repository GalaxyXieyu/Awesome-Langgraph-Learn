Focused detail

API / Queue 这一层到底做什么？

它不是简单转发请求，而是承担“控制面”的角色：接收会话上下文、参数和 graph 选择，创建 task，决定任务由谁执行，并把执行结果持续流式回传给前端。

FastAPIREST API + SSE 入口，对前端暴露 sessions / tasks / agents / models 等端点。
Task lifecycle生成 task_id，记录状态、失败、恢复点和任务输出。
Celery dispatch把长任务交给 worker，不阻塞 API 进程。
Streaming bridge把 Writer 产出的中间步骤与最终结果送回 UI。
