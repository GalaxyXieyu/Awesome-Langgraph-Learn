# html-ppt-skill 底层能力全景梳理

## 一、核心观点

**数据洞察的触发条件不应该是「给了 Excel」，而应该是「存在可分析的数据」。**

数据可以来自：
- Excel / CSV 文件
- 用户直接粘贴的 Markdown 表格
- 用户文字描述中的数字（"上个月转化率 3.1%，这个月 3.8%"）
- JSON / API 返回数据
- 图片中的表格（OCR 提取）
- 甚至用户说的「感觉很差」→ 引导用户提供具体数字

只要有数据，就自动触发 **数据洞察层**。

---

## 二、八大底层能力

### 能力 1：数据感知与读取（Data Ingestion）

**不是「Excel 读取」，是「任何结构化数据的感知」。**

| 输入形式 | 处理方式 | 工具 |
|---------|---------|------|
| Excel / CSV 文件 | pandas 读取 → DataFrame | `read_excel.py` |
| 用户粘贴的 Markdown 表格 | 正则解析 → 结构化数组 | 内置解析 |
| 文字中的数字描述 | NLP 提取 → 键值对 | 语义提取 |
| JSON 数据 | json 解析 → 结构化 | 内置解析 |
| 图片中的表格 | OCR / 视觉模型提取 | 待定（可扩展） |
| 用户口述「感觉很差」 | 反问引导 → 获取具体数字 | 对话引导 |

**关键原则**：
- 不强制要求 Excel，任何可结构化数据都接收
- 如果用户给的是模糊描述，主动引导量化（"具体数字是多少？"）
- 读取后统一输出为「标准化数据对象」（字段名 → 值数组）

---

### 能力 2：数据计算与洞察（Insight Engine）

**从原始数据 → 洞察卡片的核心引擎。**

**自动计算维度**：

| 计算类型 | 触发条件 | 示例输出 |
|---------|---------|---------|
| 同比/环比 | 存在时间序列 | "Q3 环比增长 23%" |
| 占比/构成 | 存在分类数据 | "搜索广告占投放预算 62%" |
| 均值/中位数/分布 | 存在数值列 | "客单价中位数 ¥328，均值 ¥415（右偏）" |
| ROI / 效率指标 | 存在投入/产出字段 | "ROI 1:3.2，高于行业均值 1:2.1" |
| 异常检测 | 波动超过阈值 | "11月 CTR 异常下跌 40%，需排查" |
| 相关性 | 多维度数据 | "投放预算与转化量相关性 r=0.87" |
| 排名/TopN | 存在分组数据 | "Top3 渠道贡献 78% 的转化" |

**洞察生成规则**：
- 不只是「算数字」，而是「算数字 + 给结论 + 给建议」
- 每张洞察卡片 = 断言标题 + 核心数字 + 趋势 + 上下文 + 建议
- 异常数据必须高亮（红色预警），不能淹没在表格里

---

### 能力 3：叙事编排（Narrative Orchestrator）

**把素材（用户原始内容 + 洞察卡片）编排成有故事弧线的大纲。**

**输入**：
- 用户原始素材（文字、图片、链接、需求描述）
- 数据层输出的洞察卡片数组
- 用户明确的意图（演讲/报告/混合）

**输出**：标准化大纲

```
Slide 1: Hook（封面）
  → 一句话核心结论
  
Slide 2-3: Context（背景/问题）
  → 为什么现在讲这个？数据概况
  
Slide 4-6: Build（论证）
  → 分论点 1 + 数据支撑
  → 分论点 2 + 数据支撑
  → 分论点 3 + 数据支撑
  
Slide 7-8: Proof（证据）
  → KPI 看板（洞察卡片网格）
  → 趋势图 / 对比图
  
Slide 9: Plan（行动）
  → 建议 + 时间线 / 责任人
  
Slide 10: Close（收尾）
  → 总结 + CTA / 感谢
```

**编排原则**：
- 洞察卡片不是全部堆上去，而是按叙事角色分配
- 一页一个核心论点（One idea per slide）
- 数据页和观点页交替（5/5/5 规则）

---

### 能力 4：视觉渲染（Visual Renderer）

**把大纲渲染成具体的视觉产物。**

| 渲染目标 | 技术实现 | 适用场景 |
|---------|---------|---------|
| **静态 HTML PPT** | HTML/CSS/JS + 36 主题 + 31 布局 | 演讲、技术分享、逐字稿 |
| **交互式 HTML 报告** | HTML + ECharts + 5 种报告风格 | 数据分析、周报月报、复盘 |
| **PPTX 文件** | `html2pptx.js` / `build_pptx.js` | 客户交付、需 Office 编辑 |
| **图片（PNG/JPG）** | `render.sh` (Headless Chrome) | 社交媒体、快速分享 |
| **Markdown / 文本** | 纯文本输出 | 探索性分析、快速预览 |

**关键原则**：
- 同一大纲可以渲染成多种格式（内容稳定，容器灵活）
- 渲染时自动应用对应的设计系统（主题/字体/颜色/间距）
- 图表必须是纯 SVG / 内联 JS（CDN 会导致截图白屏）

---

### 能力 5：主题与风格系统（Theme Engine）

**不是「选颜色」，是「选世界观」。**

**三层风格体系**：

| 层级 | 风格类型 | 数量 | 选择依据 |
|------|---------|------|---------|
| **PPT 主题** | 代码主题（36 套） | 36 | 受众 + 调性 + 交付方式 |
| **报告风格** | 数据报告风格 | 5 | FT / McKinsey / Economist / GS / Swiss |
| **PPT 风格** | 传统 PPT 风格 | 3 | Neo-Brutalism / Warm Narrative / Minimal |

**智能推荐**：
- 不抛 36 个主题让用户选
- 根据需求解析结果直接推荐 1 个最佳 + 1 个备选
- 说明推荐原因（"工程师+投影仪 → tokyo-night，对比度充足"）

---

### 能力 6：动画与叙事节奏（Motion Engine）

**动画不是装饰，是叙事的节拍器。**

| 动画类型 | 数量 | 用途 |
|---------|------|------|
| CSS 入场动画 | 27 | fade-up, rise-in, counter-up, stagger-list |
| Canvas FX | 20 | particle-burst, knowledge-graph, matrix-rain |

**叙事映射规则**：
- Hook → rise-in / blur-in（震撼入场）
- Build → stagger-list（逐步揭示）
- Proof → counter-up（数字跳变）
- Compare → 双向入场（左 vs 右）
- Close → 平静收束

---

### 能力 7：图片生成与处理（Image Engine）【待扩展】

**当前缺失但用户需要的能力。**

**图片在 PPT/报告中的角色**：

| 图片类型 | 生成方式 | 用途 |
|---------|---------|------|
| **封面图 / Hero Image** | AI 生成（如需要） | 视觉冲击、情绪设定 |
| **配图 / 示意图** | AI 生成 / 图标库 | 解释抽象概念 |
| **数据可视化图** | ECharts / SVG | 图表、趋势、对比 |
| **架构图 / 流程图** | 代码绘制（SVG/CSS） | 系统架构、工作流程 |
| **截图 / 产品图** | 用户提供 | 产品展示、证据 |

**与 huashu-slides 的差异定位**：
- huashu-slides：AI 图片驱动，每页都是生成的艺术图，适合视觉海报
- html-ppt-skill：代码驱动为主，图片是「辅助元素」而非「主体」
- **整合策略**：当用户需要「高质量封面图」或「概念配图」时，调用图片生成能力作为补充

**推荐实现**：
- 封面图：可使用 `nano-banana` 或 `image-this` MCP 生成
- 示意图：优先用代码绘制（SVG/CSS），确保离线可用
- 数据图表：ECharts 纯 SVG，不依赖外部服务

---

### 能力 8：演讲者支持（Presenter Support）

**不只是做 PPT，是帮用户「讲好」。**

| 功能 | 实现 | 价值 |
|------|------|------|
| 演讲者模式 | S 键弹出磁吸窗口 | CURRENT / NEXT / 逐字稿 / 计时器 |
| 逐字稿生成 | AI 根据 slide 内容生成 | 每页 150-300 字口语化脚本 |
| 计时器 | 自动计时 + 每页预估 | 控制演讲节奏 |
| 键盘导航 | ← → / Space / 数字跳转 | 演讲流畅不卡壳 |
| 预览同步 | BroadcastChannel 双窗同步 | 演示者操作实时同步到观众屏 |

## 三、数据洞察的触发条件重构

**旧逻辑（有问题）**：
```
if 用户给了 Excel:
    走数据分析路线
else:
    走纯 PPT 路线
```

**新逻辑（目标）**：
```
def 检测数据输入():
    sources = []
    if 有 Excel/CSV 附件: sources.append("excel")
    if 有 Markdown 表格: sources.append("markdown_table")
    if 文字中包含数字描述: sources.append("text_numbers")
    if 有 JSON 数据: sources.append("json")
    if 用户说"感觉很差/很好"但无数字: sources.append("need_probe")
    return sources

数据输入源 = 检测数据输入()

if 数据输入源:
    # 自动触发数据洞察层
    原始数据 = 读取并结构化(数据输入源)
    洞察卡片 = 数据计算与洞察(原始数据)
    
# 无论有没有数据，都进入叙事编排
大纲 = 叙事编排(用户素材, 洞察卡片 if exists)

# 根据交付场景选容器
容器 = 选择容器(用户需求)
最终产物 = 渲染(大纲, 容器)
```

**关键变化**：
1. 数据洞察是「自动检测」，不是「用户主动选择」
2. 没有数据也能做 PPT（纯叙事型）
3. 有数据时，数据洞察自动注入到内容中
4. 用户说"感觉转化不好"→ AI 反问"具体数字是多少？"→ 拿到数字 → 自动分析

## 五、一句话总结

**html-ppt-skill 的底层不是「PPT 制作工具」，而是「内容智能编排引擎」。**

它包含 8 大底层能力：
1. **数据感知**（任何结构化数据都能读）
2. **数据洞察**（自动计算 + 结论 + 建议）
3. **叙事编排**（Hook→Context→Build→Proof→Plan→Close）
4. **视觉渲染**（HTML PPT / 交互报告 / PPTX / 图片）
5. **主题风格**（36 主题 + 5 报告风格 + 3 PPT 风格）
6. **动画节奏**（27 CSS + 20 Canvas，映射叙事角色）
7. **图片生成**（封面/配图，AI 生成作为代码驱动补充）
8. **演讲支持**（逐字稿 + 演讲者模式 + 计时器）

**工作流**：
```
用户给需求和素材 → 自动检测数据 → 有数据则洞察 → 编排大纲 → 选容器 → 渲染 → 交付
```

数据洞察是**自动触发的底层能力**，不是**需要用户勾选的可选功能**。
